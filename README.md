# Okko Hackathon 2026 — Multilingual Video Fragment Retrieval

Этот репозиторий содержит серию экспериментов и финальное решение задачи поиска точных видеофрагментов по текстовому запросу (RU/EN). Финальная версия находится в `final_attempt/`.

**Структура**
- `attempt1/` — базовый пайплайн: Whisper Tiny + BGE‑M3 + 90s окна, простая подача фрагментов.
- `attempt2/` — резкий апгрейд качества: faster‑whisper large‑v3‑turbo, answer augmentation и HyDE (v12 ≈ 0.500).
- `attempt3/` — код DGX‑2 для v12 (тот же лучший сабмит, но в стабильной прод‑сборке).
- `attempt4/` — динамический HyDE, zero‑padding для answer‑aug, P75‑обрезка границ (v19 ≈ 0.518).
- `final_attempt/` — финальное решение: fine‑tune BGE‑M3 + доработанный поиск (v26 ≈ 0.572).
- `final_result/` — продакшен‑демо (backend + frontend + docker).

**Прогресс и различия попыток (кратко)**

Попытки `attempt1–attempt4` — это эволюция одного пайплайна; ниже более подробный маршрут улучшений и почему они сработали.

- v4 ~0.367 (attempt1): базовый поиск по 90s окнам, Whisper Tiny для ASR, BGE‑M3 без дообучения, простая выдача фрагментов.
- v8 ~0.377: переход на `faster-whisper large-v3-turbo` (8 GPU), качество транскриптов выросло, но прирост метрики был умеренный.
- v9 ~0.289 / v10 ~0.328: multi‑scale чанкинг (171K фрагментов) и coarse+fine каскад ухудшили VR@K из‑за «шума» и потери разнообразия.
- v11 ~0.470: **answer augmentation** — ответы из train добавляются как отдельные чанки с точными таймкодами. Это резко снизило семантический разрыв «вопрос ↔ лекция».
- v12 ~0.500 (attempt2/attempt3): **HyDE‑расширение** — находим похожий train‑вопрос, смешиваем эмбеддинги вопроса и его ответа (фиксированный вес).
- v15 ~0.508: **P75‑обрезка границ** — окна слишком широкие, поэтому центрируем и режем до типичной длины фрагмента.
- v19 ~0.518 (attempt4): **dynamic HyDE** (вес ответа зависит от схожести) + **zero‑padding** для answer‑aug (у них уже точные границы, padding только ухудшает IoU).
- v23 ~0.567 / v26 ~0.572 (final_attempt): **fine‑tune BGE‑M3** на train‑парах, плюс доработки в поиске и границах — финальный максимум.
